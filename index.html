Módulos 1 a 4: Fundamentos, Ethical Hacking, Reconocimiento, Scanning

📌 Módulo 1: Fundamentos de Gestión de Riesgos y SGSI Activos · Políticas · PMI

🔹 Gestión de Riesgos

Identificar, evaluar, tratar y monitorear riesgos TI para proteger CIA (Confidencialidad, Integridad, Disponibilidad).

🔹 Activos clave

Información, hardware, software, personas, servicios, infraestructura física, reputación.

🔹 Ciclo SGSI (PHVA)

Planificar (políticas, análisis), Hacer (implementar controles), Verificar (auditar), Actuar (mejora continua).

🔹 Políticas de seguridad

Documento de alto nivel con roles, principios CIA, control de accesos, gestión de incidentes y sanciones.

🔹 Análisis de riesgo

Evaluar probabilidad vs impacto. Tratamiento: mitigar, transferir, aceptar o evitar.

🔹 PMI en seguridad

Fases: Inicio, Planificación, Ejecución, Monitoreo, Cierre. Ejemplo: implementar SIEM o SGSI.

⚔️ Módulo 2: Ethical Hacking & Metodologías OSSTMM · PTES · Kali

🔹 Hacking Ético

Pruebas autorizadas para detectar vulnerabilidades antes que atacantes maliciosos.

🔹 Tipos de hackers

White Hat (ético), Black Hat (malicioso), Grey Hat (intermedio).

🔹 Fases del Ethical Hacking

Reconocimiento → Escaneo → Obtención de acceso → Mantenimiento → Cobertura de huellas → Reporte.

🔹 OSSTMM

Metodología cuantitativa, evalúa aspectos técnicos, humanos y físicos con métricas RAV.

🔹 PTES

Estándar práctico para pentesting, más técnico y ofensivo, con fases bien definidas.

🔹 Kali Linux

Distro para pentesting con Nmap, Metasploit, Wireshark, Aircrack-ng, Burp Suite preinstalados.

🕸️ Módulo 3: Reconocimiento, OSINT y Fingerprinting IA · Google Hacking

🔹 Fingerprinting

Obtener huella digital del sistema: SO, servicios, versiones. Herramienta Nmap (-O, -sV).

🔹 Reconocimiento pasivo vs activo

Pasivo: OSINT, Google Hacking, whois. Activo: escaneo de puertos, fuzzing.

🔹 OSINT y Whois

WHOIS revela propietario de dominio, DNS, fechas. Útil para mapear infraestructura.

🔹 Google Hacking (Dorks)

Operadores: site:, filetype:, intitle:"index of", inurl:admin para encontrar datos sensibles.

🔹 IA en reconocimiento

GPT para reportes automáticos, Scikit-learn para clasificar tráfico, TensorFlow para anomalías.

🔹 Shodan + Maltego

Shodan busca IoT expuestos; Maltego mapea relaciones entre dominios, correos, redes.

🔧 Módulo 4: Scanning, Enumeración y Herramientas Nmap · Netcat · Enum4linux

🔹 Scanning TCP, SYN, UDP

TCP Connect (completo), SYN (sigiloso), UDP (identifica DNS/SNMP, pero lento).

🔹 Estados de puerto

Abierto, cerrado, filtrado (firewall), unfiltered, open|filtered.

🔹 Enumeración

Usuarios (ataques fuerza bruta), servicios (versiones), software específico (CVE).

🔹 Nmap práctico

nmap -p puertos, -sV (versiones), -O (SO), -A (agresivo), -sS (SYN scan).

🔹 Netcat (navaja suiza)

Escucha/conexión, transferir archivos, shells remotas. Ej: nc -lvp 4444.

🔹 Enum4linux

Enumeración SMB en Windows: usuarios (-U), shares (-S), políticas (-o).
